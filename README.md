Единый домен данных — `live.positive-story.co.il`, всё публичное, CORS открыт (`Access-Control-Allow-Origin: *`, проверено):

Теперь живой список тянется с ЛЮБОГО домена (и даже с диска) — встроенный снимок остался только как резерв на случай недоступности API. Снимок обновлён (2026-07-09): **21 канал**.

Все счётчики «22» в статике стали «21» (в рантайме считаются из API, служебный канал `test` отфильтровывается).
- **Блог подключён к реальному эндпойнту** — `GET /api/v1/posts` реализован разработчиком (живёт на live.positive-story.co.il, а не на radio.moraltogether.com, как было в ТЗ). Пока в админке нет постов — на странице מה חדש остаются статьи-примеры; как только посты появятся, они подменятся автоматически, бейдж «בקרוב · תוכן לדוגמה» скроется. У постов без обложки (`imageUrl: ""`) подставляется фото-заглушка.
- **Тизер блога на главной** — новый: `GET /api/v1/posts?limit=4` (1 главная + 3 карточки). Пока постов нет — статичный пример, как раньше.
- **Deep-link на странице ערוצים** стал регистронезависимым: старые ссылки `#m1radio` продолжают работать с новым slug `M1radio`.
- В сетках главный канал всегда показывается первым, остальные — в порядке позиций из админки (как отдаёт API).
- **Не подключено (намеренно):** reels (`has_reels`, `GET /api/public/reels`) — ждём решения по UI; DASH-манифест (`manifest.mpd`) — HLS достаточно.

### ⚠️ Найден баг на стороне API (2026-07-09)

Сервер отдаёт заголовок `Access-Control-Allow-Origin: *` **дважды** — curl это терпит, а браузеры отклоняют такой ответ («contains multiple values '*, *'»), проверено в Chrome с чужого origin. Поэтому **фактически fetch к API из браузера с чужого домена пока не работает** — сайт в этом случае автоматически остаётся на встроенном снимке станций и статьях-примерах (так и задумано), а после исправления на сервере живые данные подхватятся сами, без правок сайта. Багрепорт разработчику: `PLAN/API-Bug-CORS.md`. Аудио/видео-потоки бага не касаются — они играют без CORS.

## Ключевая идея: дизайн от логотипа, а не от темы приложения

Web-v1 построен на теме «Vice City» (тёмный неон, маджента, свечения). Но сам бренд другой:

- Логотип — светлый, дружелюбный, ретро-поп: фиолетовый круг, розовый и голубой, винтажный микрофон.
- Слоган на логотипе — **«משדרים חיובי»** («вещаем позитив»).
- Станции — про сообщество и добро: мамы, дети с хорошими новостями, животные, ветераны, районы.
- Родительские домены: moraltogether.com, positive-story.co.il.

Поэтому эта версия — **светлая, тёплая, «человечная»**: кремовый фон, цвета из логотипа (фиолетовый / розовый / голубой), шрифт Rubik, один тёмный «остров» (Radio TV) для контраста. Слоган «משדרים חיובי» вынесен в заголовки и футер.

## Главное отличие: плеер реально работает

- Клик по любой из 22 станций запускает живой поток (`live.positive-story.co.il/<id>/shoutcast`).
- Внизу появляется плеер-док: play/pause, громкость, статусы «מתחבר…» / «באוויר» / «התחנה לא זמינה».
- У играющей станции карточка подсвечивается и аватар превращается в анимированный эквалайзер.
- Кнопки «слушать» в шапке, hero и карточках каналов тоже играют (ротация CTA по методу Timochko).

## Соответствие плейбуку

- Копирайт по методу Timochko: одна мысль на блок, позитивные формулировки, ротация CTA, eyebrow + H2 с акцентом на 1–3 словах.
- Типографика: body 18px, H1/H2 clamp, mobile-шкала одним брейкпоинтом 768px.
- Брейкпоинты: 1080 (планшет) и 768 (телефон), бургер-меню на мобильном.
- `prefers-reduced-motion` отключает анимации.
- RTL: телефонные поля `direction:ltr`.
- Никаких выдуманных фактов: без отзывов, цифр слушателей, адресов.
